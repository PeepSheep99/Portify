1. Add to api/youtube_music.py:
   - create_playlist(ytmusic, name: str, description: str = "") -> str (returns playlist_id)
   - add_tracks_to_playlist(ytmusic, playlist_id: str, video_ids: list[str]) -> int (returns count added)

2. Add POST /api/transfer endpoint to api/index.py:

   Request body:
   ```json
   {
     "oauth_token": "...",  // JSON string from OAuth flow
     "playlist": {
       "name": "...",
       "tracks": [{"name": "...", "artist": "...", "album": "..."}]
     }
   }
   ```

   Response: Server-Sent Events stream

   Implementation flow:
   1. Create YTMusic client from oauth_token
   2. Stream progress events as JSON lines with "data: " prefix:
      - {"phase": "matching", "current": 1, "total": 50, "currentTrack": "Song Name", "status": "in_progress"}
      - (one event per track during matching phase)
   3. After matching, create playlist:
      - {"phase": "creating", "status": "in_progress"}
   4. Add matched tracks in batches of 25:
      - {"phase": "adding", "current": 25, "total": 45, "status": "in_progress"}
   5. Final event with results:
      - {"status": "complete", "result": {...TransferResult...}}

   Use StreamingResponse with media_type="text/event-stream"
   Add 150ms delay between search requests (rate limiting per RESEARCH.md)
   Use duplicates=True when adding tracks (per RESEARCH.md pitfall #5)
  </action>
